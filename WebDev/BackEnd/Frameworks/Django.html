Django

Definition

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. It follows the Model-View-Template (MVT) architecture, provides built-in security features, and simplifies tasks like URL routing, database management, authentication, and template rendering.

Important Django Concepts

Core Concepts

  • Project & App Structure (django-admin startproject, startapp)
  • Models (models.Model) – Database representation
  • Views (def or class-based views) – Logic for handling requests
  • Templates (.html files) – Rendering dynamic content
  • URL Routing (urls.py) – Mapping URLs to views
  • Admin Interface (admin.site.register) – Built-in admin panel
  • Forms (forms.Form, forms.ModelForm) – Handling user input
  • QuerySets & ORM (.objects.all(), .filter()) – Database queries
  • Middleware – Request/response processing
  • Settings (settings.py) – Configurations for project

Advanced / Important Topics

  • Class-Based Views (CBV) – ListView, DetailView, etc.
  • Authentication & Authorization – User model, login, permissions
  • Static Files & Media – CSS, JS, images management
  • Template Tags & Filters – {% %} and {{ }} syntax
  • Signals – Hook into model events
  • Pagination – Splitting query results into pages
  • REST Framework (Django REST Framework) – API development
  • Deployment – WSGI, ASGI, Gunicorn, Nginx
  • Testing – Unit tests with TestCase
  • Security Features – CSRF, XSS prevention, password hashing
