MongoDB

Definition

MongoDB is a NoSQL, document-oriented database that stores data in JSON-like BSON documents. It is designed for high performance, scalability, and flexibility, and it does not require a fixed schema like traditional relational databases.

Important MongoDB Concepts

Core Concepts

  • Databases & Collections – Organize data in databases and collections
  • Documents – JSON-like data records
  • _id – Unique identifier for each document
  • CRUD Operations – Create, Read, Update, Delete
  • Queries – find(), findOne(), filters
  • Projection – Selecting specific fields in a query
  • Indexes – Improve query performance
  • Aggregation – $match, $group, $sort for data analysis
  • Data Types – String, Number, Boolean, Array, Object, Date
  • Embedded Documents & Arrays – Nested structures for flexible data

Advanced / Important Concepts

  • Replication – Replica sets for high availability
  • Sharding – Horizontal scaling of data
  • Transactions – Multi-document ACID transactions
  • Validation Rules – Enforce data integrity
  • MongoDB Atlas – Cloud database service
  • Indexes Types – Single, compound, text, geospatial
  • Aggregation Pipeline – Advanced data processing
  • Map-Reduce – Custom data processing functions
  • Change Streams – Real-time notifications on data changes
  • Backup & Restore – Data safety and recovery
